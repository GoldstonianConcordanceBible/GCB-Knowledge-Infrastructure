# GitHub Credit Policy
## SydTek University

SydTek University uses GitHub as a primary evidence environment for many credit-bearing courses, projects, studios, and capstones.

## Core Rule

Students receive academic credit only when their work is documented through verifiable GitHub activity and linked course artifacts, unless a course explicitly states an approved alternative evidence system.

## Minimum Evidence Standard

To receive credit in GitHub-based courses, students must provide academic evidence that may include:

- commits
- version history
- dated artifact development
- issue tracking where applicable
- pull requests where applicable
- linked reflections
- linked written submissions
- linked presentations
- linked rubric-scored deliverables

## Public or Private Repository Choice

Repositories may be:

- public
- private

The choice may be made by:

- the student
- the parent or guardian when applicable
- the institution when required by course structure

## Verification Requirement

Regardless of repository visibility, faculty or authorized academic reviewers must be able to verify:

- authorship
- progression of work
- submission timing
- completeness of required artifacts
- alignment with course outcomes

## Credit Principle

Credit is awarded for verified learning and documented performance, not merely for repository creation or public posting.

## Equivalent Rigor

If public participation is encouraged in a course, students who use private repositories must still complete an equivalent rigorous pathway with comparable:

- workload
- evidence standards
- assessment expectations
- reflection requirements
- revision requirements

## Instructor Authority

Course instructors may define additional GitHub submission requirements in the syllabus, including:

- branch naming rules
- commit frequency expectations
- milestone checkpoints
- repository organization standards
- documentation formats

## Preservation

Important student artifacts should be preserved through institutional evidence dossiers, audit packets, or approved archival methods where appropriate.
