# Future File Rule
## SydTek University

This file defines a standing rule for future SydTek University course, program, and policy files.

## Rule 1

Future course syllabi, capstone documents, practicum documents, and assessment files should state whether GitHub-based evidence is required.

## Rule 2

Where GitHub-based evidence is required, files should clarify that:

- academic credit depends on verifiable documented work
- repositories may be public or private unless the course explicitly requires public engagement
- authorized reviewers must be able to inspect required evidence
- equivalent rigorous pathways must exist when public participation is not appropriate

## Rule 3

Where peer review is included, files should distinguish between:

- public peer review
- private peer review
- equivalent critique pathways

## Rule 4

Where Open Discussion Pedagogy is used, files should align prompts, rubrics, and reflection tasks with:

Mirror → Water → Fire

## Rule 5

Future files should preserve the stable institutional names:

- SydTek University
- SydTek Scholars
- Goldstonian Concordance Bible
- Mirror → Water → Fire Doctrine
