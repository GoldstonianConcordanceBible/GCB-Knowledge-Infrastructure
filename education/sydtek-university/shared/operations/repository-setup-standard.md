# Repository Setup Standard
## SydTek University

This document defines a baseline setup standard for student repositories in GitHub-based courses.

---

## Purpose

A consistent repository setup makes academic review easier and improves documentation quality.

---

## Recommended Basic Structure

A student repository may include:

- README.md
- assignments/
- reflections/
- artifacts/
- drafts/
- final/
- evidence-notes/

---

## Naming Principle

Repository and folder names should be clear, stable, and review-friendly.

---

## Documentation Principle

Students should provide enough documentation so an authorized reviewer can understand:

- what the repository is for
- what major artifacts are included
- what work was completed
- where the final deliverables are located

---

## Visibility Principle

Repositories may be public or private unless the course explicitly requires otherwise.

Either way, authorized academic reviewers must be able to inspect the required evidence.
