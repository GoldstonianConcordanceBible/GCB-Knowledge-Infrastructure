# Audit Packet Workflow
## SydTek University

This workflow explains how audit packets should be assembled for course and program review.

---

## Purpose

The audit packet organizes course-level or program-level documentation into one review-ready structure.

---

## Workflow Steps

### Step 1 — Gather Core Academic Documents
Include:

- syllabus
- course shell
- assignment map
- rubric set
- assessment summary

### Step 2 — Gather Evidence Samples
Include selected evidence dossiers, exemplar artifacts, and evaluation records.

### Step 3 — Add Review Notes
Include:

- instructor notes
- calibration notes
- improvement notes
- moderation notes where applicable

### Step 4 — Add GitHub Evidence References
Where GitHub is used, include public or private repository references, snapshots, or exports sufficient for authorized review.

### Step 5 — Preserve the Packet
Store the audit packet in the approved institutional evidence location.

---

## Review Standard

A complete audit packet should allow a reviewer to understand:

- what the course or program intended
- what students were asked to do
- how work was assessed
- what evidence exists
- what improvements were identified
