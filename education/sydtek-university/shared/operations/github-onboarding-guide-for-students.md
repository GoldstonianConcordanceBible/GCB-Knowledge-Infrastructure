# GitHub Onboarding Guide for Students
## SydTek University

This guide introduces students to GitHub-based academic work at SydTek University.

---

## Why GitHub Is Used

Many SydTek courses use GitHub because it helps preserve:

- work progression
- documentation
- versions
- milestone evidence
- final artifacts

GitHub is used as an academic evidence environment.

---

## Core Rule

If your course is GitHub-based, academic credit depends on verifiable documented work aligned to course outcomes.

Your repository may be public or private unless the course states otherwise, but authorized academic reviewers must be able to inspect the required evidence.

---

## What Students Should Learn to Do

Students should learn how to:

- create and organize repositories
- make meaningful commits
- preserve milestone progress
- store documentation clearly
- submit repository links or approved exports
- respond to critique and revise work

---

## Public and Private Choice

Depending on the course and your circumstances, your repository may be:

- public
- private

Public work may help with portfolio building.

Private work is acceptable when permitted, provided faculty can review it.

---

## Good Student Practice

Strong GitHub-based academic practice includes:

- clear folder names
- regular commits
- accurate file organization
- dated progress
- short documentation notes
- final clean submission state

---

## Important Reminder

Creating a repository alone does not earn credit.

Credit comes from documented learning, complete work, and faculty-verifiable evidence.
