# WEB3-301 Course Shell
## Smart Contracts and Decentralized Applications

Credits: 3  
Level: Undergraduate

## Catalog Description

Examines smart contracts, programmable execution, decentralized applications, wallet interaction, contract state, and application-layer design.

## Prerequisites

WEB3-201  
WEB3-220

## Course Learning Outcomes

Students will be able to:

1. explain the role of smart contracts in Web3 systems
2. describe the structure of decentralized applications
3. analyze wallet, contract, and interface interaction
4. identify basic risks in contract-based design
5. document application reasoning and artifact development

## GitHub Evidence Statement

Where GitHub-based evidence is required, academic credit is awarded for verifiable documented work aligned to course outcomes. Repository visibility may be public or private unless otherwise stated, but authorized academic reviewers must be able to inspect required evidence.

## Open Discussion Pedagogy Note

This course may include critique and revision cycles around dApp design and contract use cases.

## Mirror → Water → Fire Alignment

Mirror:
represent the contract logic and use case accurately

Water:
interpret the application context and system interaction

Fire:
critique, refine, and synthesize an application design
