# Assessment Plan
## BS in Web3 Systems and Decentralized Infrastructure

Institution: SydTek University

This document defines how SydTek University will evaluate student learning and program effectiveness.

---

## Assessment Philosophy

Assessment should demonstrate:

- student learning
- curriculum coherence
- program quality
- improvement over time

Assessment in this program uses direct and indirect measures.

Assessment may include public or private evidence pathways, but all pathways must remain academically equivalent in rigor and documentation.

---

## Direct Measures

Direct measures include:

- exams
- technical exercises
- labs
- case analyses
- project artifacts
- GitHub repository submissions
- capstone deliverables
- portfolio reviews
- evidence dossiers

---

## Indirect Measures

Indirect measures include:

- student surveys
- course evaluations
- exit interviews
- alumni feedback
- internship reflections

---

## GitHub-Based Evidence Rule

In courses designated as GitHub-based, students receive credit only when faculty can verify learning through documented repository activity and linked course artifacts.

Repository visibility may be public or private.

The academic requirement is verifiable evidence, not public exposure.

---

## Public and Private Pathways

When courses encourage open discussion or public peer review, students must still have access to an equivalent rigorous pathway where legitimate privacy, age, safety, accessibility, or family considerations apply.

Equivalent pathways must preserve:

- learning outcomes
- critique engagement
- reflection
- revision
- rubric-based evaluation
- documentation quality

---

## Program-Level Assessment Schedule

### Annual Review
Each academic year the program will review:

- student achievement on program learning outcomes
- pass rates in major core courses
- quality of final projects
- faculty observations
- curriculum gaps

### Three-Year Review
Every three years the program will review:

- curriculum relevance
- sequencing effectiveness
- employer and industry alignment
- portfolio quality
- benchmark comparisons

---

## Assessment Responsibility

Assessment responsibilities are distributed across:

- program lead
- course instructors
- curriculum committee
- academic quality review team

---

## Capstone Benchmark Areas

Capstone projects will be evaluated for:

- technical accuracy
- systems integration
- governance logic
- security awareness
- communication quality
- originality and feasibility
- evidence quality
- documentation integrity

---

## Improvement Loop

Assessment findings will be used to:

- revise course outcomes
- strengthen assignments
- improve sequencing
- add support resources
- refine capstone expectations
- improve evidence preservation practices
