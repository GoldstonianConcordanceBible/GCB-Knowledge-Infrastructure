# Capstone Model
## BS in Web3 Systems and Decentralized Infrastructure

Institution: SydTek University

This document outlines the culminating experience for the degree.

---

## Purpose

The capstone demonstrates a student’s ability to integrate the major areas of the program into one coherent project.

---

## Capstone Expectations

A capstone project should show:

- problem definition
- technical design
- governance logic where relevant
- security awareness
- documentation quality
- professional presentation

---

## Possible Capstone Formats

Students may complete:

- decentralized application prototype
- token economy design
- governance system model
- blockchain case analysis with architecture proposal
- machine-readable educational repository
- applied Web3 research project

---

## Required Deliverables

Capstone students should submit:

- project proposal
- milestone updates
- final artifact
- written report
- presentation
- portfolio-ready repository

---

## Evaluation Criteria

Capstone evaluation should consider:

- technical rigor
- integration of course knowledge
- communication quality
- originality
- feasibility
- professionalism
