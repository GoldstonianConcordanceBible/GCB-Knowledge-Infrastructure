# Capstone Model
## BS in Web3 Systems and Decentralized Infrastructure

Institution: SydTek University

This document outlines the culminating experience for the degree.

---

## Purpose

The capstone demonstrates a student’s ability to integrate the major areas of the program into one coherent project.

---

## Capstone Expectations

A capstone project should show:

- problem definition
- technical design
- governance logic where relevant
- security awareness
- documentation quality
- professional presentation
- verifiable development evidence

---

## GitHub Requirement

Capstone students must complete the project in a GitHub repository or approved equivalent version-controlled environment.

The repository may be public or private unless the specific capstone track requires public engagement.

Credit is awarded only when authorized reviewers can verify:

- authorship
- progression
- milestone completion
- final artifact integrity

---

## Possible Capstone Formats

Students may complete:

- decentralized application prototype
- token economy design
- governance system model
- blockchain case analysis with architecture proposal
- machine-readable educational repository
- applied Web3 research project

---

## Required Deliverables

Capstone students should submit:

- project proposal
- milestone updates
- final artifact
- written report
- presentation
- portfolio-ready repository or archived equivalent
- evidence dossier

---

## Evaluation Criteria

Capstone evaluation should consider:

- technical rigor
- integration of course knowledge
- communication quality
- originality
- feasibility
- professionalism
- evidence quality
