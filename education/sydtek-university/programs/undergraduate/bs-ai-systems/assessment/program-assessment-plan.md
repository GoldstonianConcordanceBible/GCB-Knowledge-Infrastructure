# BS-AIS Program Assessment Plan
Bachelor of Science in Artificial Intelligence Systems

Institution: SydTek University

## Purpose

This document explains how the BS-AIS program assesses student learning, reviews program effectiveness, and preserves inspectable evidence for academic quality and accreditation-oriented review.

The program uses course-embedded assessment, GitHub-verifiable artifacts, faculty review, and capstone evaluation to determine whether students are meeting program outcomes.

## Assessment Philosophy

Assessment in the BS-AIS program is based on documented student performance rather than only seat time or exam exposure.

Where GitHub-based assessment is used, repositories serve as inspectable evidence environments that allow faculty and authorized reviewers to examine:

- process
- revision
- technical execution
- documentation quality
- reproducibility
- final outcomes

The assessment model aligns with the broader SydTek University emphasis on evidence-linked learning and documented academic work.

## Program Learning Outcomes Assessed

Graduates of the BS-AIS program should be able to:

1. explain foundational concepts in artificial intelligence, machine learning, neural networks, and intelligent systems
2. apply mathematics, statistics, and computational reasoning to AI problems
3. build, evaluate, and refine machine learning and deep learning models
4. design reproducible AI workflows using documented datasets, code, and evaluation methods
5. analyze ethical, social, and governance issues related to AI systems
6. communicate AI findings through technical writing, repository documentation, and inspectable artifacts
7. design AI systems that integrate models, infrastructure, and deployment considerations
8. complete a substantial capstone project demonstrating undergraduate-level mastery in AI systems

## Direct Assessment Sources

Direct assessment may include:

- exams and quizzes
- technical notebooks
- coding assignments
- data workflow artifacts
- model training and evaluation reports
- architecture analyses
- ethical analysis papers
- presentation materials
- capstone repositories
- faculty-scored rubrics

## Indirect Assessment Sources

Indirect assessment may include:

- student reflections
- course feedback
- exit surveys
- alumni feedback
- internship or practicum feedback where applicable
- faculty curriculum review notes

## GitHub-Based Evidence Model

In courses using repository-based assessment, faculty may review evidence such as:

- README files
- code and notebooks
- data preparation documentation
- issue logs
- commit histories
- pull requests
- model cards
- experiment logs
- evaluation summaries
- release notes
- final reports
- presentation files

GitHub evidence does not replace faculty judgment. It strengthens academic review by making learning inspectable.

## Assessment Schedule

### Annual Cycle

Each academic year, the program should complete the following cycle:

1. collect selected course-level evidence tied to program outcomes
2. review capstone artifacts and faculty scoring patterns
3. identify strengths, weaknesses, and recurring gaps
4. document actions for curricular improvement
5. preserve an annual assessment summary for audit and accreditation review

### Multi-Year Rotation

A multi-year rotation may be used so that every program outcome is directly reviewed on a scheduled basis, with capstone evidence serving as a recurring integrative checkpoint.

## Performance Benchmarks

Suggested program benchmarks may include:

- a defined percentage of students meeting or exceeding expectations on outcome-aligned rubrics
- successful completion of required capstone milestones
- acceptable performance in repository documentation quality
- evidence of reproducibility in designated courses
- evidence of ethical analysis in governance-related coursework

Benchmarks should be reviewed periodically by faculty and adjusted when needed.

## Faculty Review Process

Faculty review may include:

- course-level artifact sampling
- rubric-based scoring
- capstone panel review
- curriculum alignment review
- assessment meeting notes
- documented action items for improvement

## Capstone as Integrative Assessment

AIS-495 serves as a culminating demonstration of program achievement.

The capstone should be used to assess whether students can integrate:

- AI concepts
- data workflows
- modeling logic
- evaluation methods
- technical documentation
- ethical reasoning
- presentation and communication

## Continuous Improvement

Program assessment results should inform:

- course revisions
- prerequisite adjustments
- rubric refinement
- additional student support
- changes to evidence expectations
- updates to capstone standards
- curriculum sequencing improvements

## Documentation for Review

The program should preserve assessment records sufficient for internal review, audit support, and accreditation preparation.

These records may include:

- annual assessment summaries
- selected anonymized student artifacts where permitted
- rubric templates
- faculty review notes
- curriculum revisions linked to assessment findings

## Mirror → Water → Fire Assessment Framing

Mirror: determine whether students represent concepts, systems, and evidence accurately  
Water: determine whether students interpret results, tradeoffs, and implications responsibly  
Fire: determine whether students can refine, document, present, and complete substantial AI work

## Closing Statement

The BS-AIS assessment plan is designed to support rigorous undergraduate review through direct evidence, inspectable technical artifacts, faculty oversight, and continuous curricular improvement.
