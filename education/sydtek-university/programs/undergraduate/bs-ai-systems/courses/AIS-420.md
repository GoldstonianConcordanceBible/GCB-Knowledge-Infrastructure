AIS-420 Course Shell
AI Security

Credits: 3  
Level: Undergraduate

Catalog Description

Introduces security issues in artificial intelligence including adversarial inputs, data poisoning, model theft, inference risk, privacy concerns, system misuse, and operational safeguards.

Students evaluate how AI systems can fail or be exploited and how technical and organizational controls can reduce risk.

Prerequisites

AIS-320  
AIS-340

Course Learning Outcomes

Students will be able to:

1 explain major security risks affecting AI systems
2 analyze adversarial, privacy, and model integrity threats
3 evaluate how infrastructure and governance decisions influence AI security
4 compare technical and procedural mitigation strategies
5 document security analysis and recommendations through inspectable academic artifacts

GitHub Evidence Statement

Where GitHub-based evidence is used, academic credit is awarded for verifiable documented work aligned to course outcomes.

Repository visibility may be public or private unless otherwise stated, but authorized academic reviewers must be able to inspect required evidence.

Open Discussion Pedagogy Note

This course may use structured critique of AI system vulnerabilities, model misuse scenarios, and institutional risk decisions.

Mirror → Water → Fire Alignment

Mirror: represent threat models, system components, and security concepts accurately  
Water: interpret vulnerabilities, risks, and mitigation tradeoffs responsibly  
Fire: produce documented AI security analysis and system hardening recommendations
