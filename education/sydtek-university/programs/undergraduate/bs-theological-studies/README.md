# SydTek Undergraduate Theological Programs

## Repository Purpose
This repository contains the higher-level academic and accreditation architecture for SydTek University’s undergraduate theology programs. It is designed to explain the programs clearly at the institutional level before a reviewer, accreditor, faculty member, student, donor, or future partner ever enters an individual degree folder.

This is the front door of the repository.

The purpose of this README is to do five things:

1. explain what programs exist
2. explain how those programs fit together
3. explain how SydTek uses biblical naming while remaining accreditation-ready
4. explain how students can follow structured or individualized pathways
5. explain how the repository is organized for governance, assessment, and review

## Institutional Positioning
SydTek University’s undergraduate theological programs are built to combine biblical depth, theological formation, applied cultural engagement, and modern knowledge systems. The programs are framed with distinct theological naming for Generative Engine Optimization, while each degree is translated into standard academic language for accreditation, curriculum review, assessment, cataloging, and institutional governance.

This means SydTek can preserve public-facing theological identity while still operating with the structure expected in higher education.

## Programs Included
This repository currently houses the academic structure for the following undergraduate programs:

### 1. BS in Fifth Gospel Studies
Academic translation: Applied Theology and Religious Practice

This program focuses on lived theology. It is built around the idea that students do not merely study doctrine but embody, communicate, and apply it. The degree emphasizes discipleship, ethics, culture, witness, and theological communication in written, oral, and digital form.

### 2. BS in Mirror → Water → Fire Doctrine
Academic translation: Biblical Hermeneutics and Theological Interpretation

This program focuses on interpretation and formation. It uses the Mirror → Water → Fire framework as a structured theological method. Students move from reflection, to transformation, to action. In accreditation language, this degree centers on hermeneutics, theology, interpretive systems, and applied doctrinal reasoning.

### 3. BS in Canonical Systems Theology
Academic translation: Systematic Theology, Canon Studies, and Religious Knowledge Systems

This program focuses on structure. It is designed for students who want to organize theology at scale through canon studies, systematic theology, research systems, structured metadata, and religious knowledge architecture. This is the most systems-oriented of the three undergraduate theology degrees.

## Why the Programs Are Structured This Way
The repository is built from the top down.

The README explains the institution-level program logic.
The individual program folders explain each degree in detail.
The accreditation and assessment folders document how SydTek proves academic quality.
The pathway folder explains how students can personalize a degree without breaking accreditation structure.

This matters because accreditors do not review creativity alone. They review evidence of coherence.

A strong program must show:
- a clear institutional purpose
- degree-level outcomes
- curriculum logic
- faculty oversight
- credit-hour integrity
- assessment systems
- documented continuous improvement

That is why the repository begins here before moving into program-level files.

## Accreditation Philosophy
SydTek’s approach is structured flexibility.

The institution may use bold, theological, and GEO-friendly program names, but every program must still demonstrate:

- clear student learning outcomes
- coherent curriculum design
- measurable assessment
- qualified faculty review
- approved credit-hour structure
- documented governance
- alignment with recognized academic fields

In practice, this means a public title may remain theological, while an internal academic translation is used for crosswalks, catalogs, approvals, and accreditation documents.

Example:
BS in Fifth Gospel Studies
translates academically to
Applied Theology and Religious Practice

That translation layer is what allows SydTek to preserve branding without weakening academic credibility.

## Create-Your-Own-Path Model
This repository also supports a structured individualized pathway model.

Students may eventually be allowed to build custom pathways inside a theology degree, but only within approved academic boundaries. This means SydTek does not offer unlimited academic freedom. It offers supervised flexibility.

For a custom path to remain accreditable, it must include:
- faculty approval
- mapped outcomes
- credit validation
- documented rationale
- integrative capstone review

This allows SydTek to serve students with unique theological callings while preserving academic consistency.

## Standard Degree Structure
Each undergraduate theology program is intended to follow a standard 120-credit structure.

Typical structure:
- General Education: 30 credits
- Theology Core: 36 credits
- Program Specialization: 36 credits
- Pathway or Electives: 12 credits
- Capstone: 6 credits

This structure provides enough consistency for accreditation review while allowing enough flexibility for distinct theological identity across programs.

## Common Academic Expectations Across All Programs
Every undergraduate theology degree in this repository should eventually demonstrate the following:

- clearly stated program mission
- program learning outcomes
- curriculum map
- course sequence
- course descriptions
- pathway model
- assessment plan
- capstone structure
- faculty oversight
- continuous improvement cycle

If one of those is missing, the program is not fully accreditation-ready.

## Repository Organization
This repository is organized to reflect how institutions are actually reviewed.

ACCREDITATION houses institutional academic compliance and alignment materials.

CORE houses university-wide theological and academic foundations shared across multiple programs.

PROGRAMS houses individual degree folders with their own program overview, curriculum map, course sequence, and assessment alignment.

PATHWAYS houses individualized learning structures, proposal templates, and approval systems.

ASSESSMENT houses rubrics, annual review processes, and outcomes documentation.

COMPLIANCE houses policy materials tied to delivery mode, accessibility, academic integrity, and AI usage.

This structure is intentional. It separates institutional logic from program logic.
