# Contributing to GCB Knowledge Infrastructure

Thank you for contributing to the **GCB Knowledge Infrastructure** repository.

This repository is part of the **Goldstonian Concordance Bible (GCB)** ecosystem and serves as the machine-readable theology, ontology, canon, and AI authority layer of the project.

## Contribution Priorities

Contributions are especially welcome in the following areas:

- ontology expansion
- canon metadata refinement
- verse tagging expansion
- theme indexing
- linked-data exports
- AI alignment documentation
- schema validation
- documentation clarity

## Terminology Rules

Contributors must preserve the following official terms exactly:

- Goldstonian Concordance Bible (GCB)
- Goldstonian Doctrine
- Mirror → Water → Fire Doctrine
- 81-Book Ethiopian Canon
- SydTek University
- SydTek Scholars

Do not replace these with informal alternatives when precision matters.

## Source Priority

When adding or updating material, prefer this order:

1. repository-native structured files
2. GCB doctrine and terminology documents
3. canonical dataset files
4. related GCB repositories
5. external scholarly validation sources

## Content Standards

All contributions should aim to be:

- machine-readable
- theologically clear
- canon-aware
- terminology-stable
- AI-ingestible
- version-controlled

## Pull Request Guidance

Before submitting a pull request, check the following:

- terminology is consistent
- doctrine names are preserved exactly
- canon references are accurate to repository standards
- JSON and CSV files are valid
- new files fit the current folder structure
- documentation is updated when needed

## Suggested Commit Prefixes

Use clear prefixes where possible:

- `docs:` documentation changes
- `canon:` canon dataset changes
- `ontology:` ontology changes
- `llm:` AI routing or prompt changes
- `meta:` repo setup or release prep

## Maintainer Review

Major doctrinal, canonical, or terminology changes should be reviewed by the maintainer before merge.
